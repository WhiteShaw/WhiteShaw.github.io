矢量网络分析仪自动化软件开发流程

Posted on 2019-03-14 | In 杂谈类

一、起源

​ 想想已经很久没有写过文章了，可能上次正式写的文章还是在QQ空间中。自从使用微信之后，就只会发一些图片以及简短文字。长时间不写文章、不上台发言，人的思维都会变得迟钝。因此想要锻炼思维，二者得取其一。由于现在上台发言的机会甚少，只能通过写文章来锻炼自己的思维。

​ 前两天在网上搜：如何结识优秀的人。其中看到一个观点觉得很正确。那就是想要结识优秀的人，首先要让自己变得优秀，要不断学习提高自己，并且要分享出来。为啥呢，因为优秀的人更愿意结交优秀的人，不仅如此，你还要让别人知道你是优秀的人，或者是一个愿意变优秀的人，因此还要学会分享。我认为写文章刚好是一个一石二鸟的方法。不但能巩固自己学到的东西，也是一种分享途径。

​ 回归正题。这段时间我一直想着是不是要转行不做研发了。因为我觉得在安全这个行业，研发能做的事情比较少，而主要在销售。而且我还产生一个想法就是，要想挣更多的钱，就必须要做更多靠近用户，靠近市场的事情，这些都是决定你能挣多少钱的直接因素，而不是新开发了一个什么功能，或者修改了一个什么bug。或者说相对来说前者更加重要。但是我也不敢说直接就转行做销售，也不知道真正做销售是什么样子的，因此我就想找个人了解一下。我就想到了大牙，我的大学同学，前段时间转行做销售了。然而和他聊了一阵之后，他说他现在正在销售的矢量网络分析仪，做它的自动化测试软件有一定市场。因此我们一拍即合，准备来做这么一套东西。

二、调研

​ 决定之后，我就开始了解，这个矢量网络分析仪到底是个什么东西。看了一些资料之后，发现这个东西真的是一个很复杂的东西。再加上大学的时候像物理，数学，信号处理等基础课程都没学好，很多东西更难理解。要想全面了解其原理真的是很难，但是因为我只是做自动化测试软件，在没有完全了解之前也能先开始做。因此我就着手于怎么使用上，到目前为止，这个项目已经开始快一个月了，我对它的了解就是一个测试仪，能够测试像线缆参数这样的一个东西。而手动操作也仅限于测试线缆的S11参数。不过也只能先往后做，期望以后会有一个更加全面的了解。前两天和大牙聊天的时候我又问他这个东西什么时候能挣钱，大牙可能觉得我有些急躁。后来我想了下，可能确实是我比较浮躁了，因为最近老想着怎么挣钱，所以变得很浮躁。静下来思考一下也确实是不对，即使想挣钱也不是这么浮躁能成功的。而是应该着眼于怎么提高自己，使自己变得更优秀，然后去结识更优秀的人，看别人都是怎么挣钱的。以我现在的能力可能想破脑袋也不能挣更多的钱。所以我应该放慢脚步，不断学习，不断思考，向优秀的人学习，才能达到目标。因此我不需要这么着急着要把这个东西做出来，而是慢工出细活，每天进步一点。并在这个过程中不断提升自己。关键是要坚持，不要一鼓作气，然后放弃。

略

四、转变

​ 本来已经把第三章写了很多内容，但是今天早上来上班的时候突然相通了一个问题。其出发点就是思考我昨晚所想的做事情要放慢脚步，不要急躁这个观点。我突然觉得昨晚的那个想法是不正确的。我认为自我提升也应该和工作中做任务是一样的道理，必须要先有目标，然后有计划，然后是执行，最后检查完成情况，我简称为四要素。如果做一个事情，只是想着有空了就做一做，没空了就不做的心态，那么这件事十有八九是做不好的。首先就是目标都不够清晰，如何把事情做好，第一步都做不好。因此我决定改变之前的策略。

​ 具体到这个事情上就是，我虽然决定不了大牙是否能在市场上取得什么收获。但是我可以在开发上面决定我的那四要素。因此我应该制定目标，以尽快速度完成任务。我就不能采取之前的态度：东西慢慢做，做一些就写博客，不用在意速度。

​ 因此我果断删掉第三章，计划不再将技术内容写下来，因为那些东西完全可以在代码中体现，没必要再在博客中写一次，这也是为了节省时间。接下来就是确定这个项目四要素了。

1. 目标

​ 开发一套软件，能够完成适量网络分析仪针对线缆的S11测试的自动化测试及结果展示和判定。具体到功能上就是3个界面，连接，配置，结果展示。以及license控制功能。

2. 计划

​ 目前已经完成连接的功能，配置的功能刚开始。计划是在测试仪能用的这段时间，也就是下周4之前，将所有需要用到的功能的命令实现方式掌握，包括返回数据的格式。然后再利用2周的时间完成界面的开发，其主要工作量在于结果展示上。最后2周时间完成打包及license功能。

Posted on 2019-03-14 | In 技术类

问题：

1. centos6 的版本比较老，安装nodejs按照教程不好使，使用如下命令：

sudo rpm -ivh http://download.fedoraproject.org/pub/epel/6/x86_64/epel-release-6-8.noarch.rpm
sudo rpm –import /etc/pki/rpm-gpg/RPM-GPG-KEY-EPEL-6
sudo yum -y install nodejs npm –enablerepo=epel

2. 如果服务器的端口不是标准的22，则需要进行以下修改：

cat>~/.ssh/config

host newdomain
hostname domain.com
port 3022
